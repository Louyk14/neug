# 结果分析与可视化 (Result Analyze)

> 本文件是 SKILL.md Step 4.3-4.7 的详细补充指南。核心规则（数值引用、必须可视化、递归总结）已在 SKILL.md 中内联，此处提供详细绘图代码和格式模板。

对查询结果进行分析，提取结论，绘制可视化图表，并在多个子问题之间进行综合分析。

## Part 1: 单子问题结果分析

对每个叶子子问题的查询结果：

### 1.1 提取关键结论

所有结论**必须引用具体数值**：

| 禁止 | 正确 |
|------|------|
| "成绩较高" | "平均成绩为 85.3 分" |
| "数量较多" | "共有 1,234 所学校" |
| "增长明显" | "同比增长 23.5%" |

从结果中提取：
1. 核心数据指标
2. 排名/分布特征
3. 异常值或突出数据点
4. 与预期的差异

### 1.2 绘制可视化

每个子问题的分析**必须**有可视化。使用 Python (matplotlib/seaborn) 绘图。

**⚠️ 环境准备（首次绘图前必须执行一次）**:

```bash
python3 -m venv myvenv
source myvenv/bin/activate
pip install matplotlib seaborn
```

环境创建后，后续每次绘图前只需 `source myvenv/bin/activate` 激活即可。

**图表类型选择**:

| 数据特征 | 推荐图表 | 使用场景 |
|----------|----------|----------|
| 类别比较 | 柱状图 | 不同组之间的数值对比 |
| 时间趋势 | 折线图 | 数据随时间的变化 |
| 占比分布 | 饼图/环形图 | 各部分占总体的比例 |
| 两变量关系 | 散点图 | 相关性分析 |
| 多组对比 | 分组柱状图 | 多维度交叉对比 |
| 分布特征 | 箱线图/直方图 | 数据分布形态 |

**绘图规范**:

## Part 2: 综合分析（递归总结）

同级子问题全部完成后，撰写总结小节。

### 总结要求

1. **综合分析**: 不是简单罗列子问题结论，而是发现关联和规律
2. **交叉对比**: 对比不同子问题的发现，寻找一致性或矛盾
3. **引用数据**: 总结中引用各子问题的具体数据
4. **局限说明**: 指出分析的局限性和待探索方向

### 总结报告格式

```markdown
### X.(k+1) 总结: [上级问题标题]

**综合分析**:
根据 X.1 ~ X.k 的分析结果，对于"[上级问题]"：
1. [综合结论1，引用子问题数据]
2. [综合结论2，对比不同子问题发现]

**子问题间的关联**:
- [关联描述，如"Q1 的发表量下降与 Q2 的合作减少互相印证"]

**局限与展望**:
- [分析局限和后续方向]
```

### 递归规则

```
完成子问题 Q.1, Q.2, ..., Q.k
    │
    ▼
增加 Q.(k+1) 总结，综合回答 Q
    │
    ▼
如果 Q 的兄弟问题也全部完成
    │
    ▼
在上一级增加总结，综合回答更上级问题
```

最终：所有第一级子问题完成后，增加**顶层总结**回答用户原始问题。
