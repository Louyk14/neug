# 深入分析 (Deep Analysis)

> 本文件是 SKILL.md Step 4.8 的详细补充指南。核心触发条件已在 SKILL.md 中内联，此处提供详细的分析方向和迭代策略。

在基础分析和对照分析完成后，判断当前结果是否存在值得进一步探索的点，并迭代处理。

## 1. 深入分析触发检查

完成所有第一层子问题的基础分析和对照后，**必须**逐一检查以下维度：

| 检查维度 | 判断方法 | 优先级 | 示例 |
|----------|----------|--------|------|
| **原始问题未回答** | 用户核心问题未被当前分析覆盖 | ⭐⭐ 最高 | 问"资金与出版物关系"但因数据缺失转而分析了其他内容 |
| **可查的局限/建议** | 4.7 总结中标记为"可转化为子问题"的局限/建议 | ⭐⭐ 最高 | "建议分析合作网络" → 可查 → 必须执行 |
| **第二层映射** | Step 2 中标记为"待深入探索"的候选映射 | ⭐ 高 | 引用指标已分析，质量维度（journal_rank）待探索 |
| 异常值 | 某个数据点显著偏离整体分布 | 高 | Top1 的值是 Top2 的 3 倍以上 |
| 意外发现 | 结果与常识或假设相悖 | 高 | 预期正相关但实际负相关 |
| 对照异常 | 对照分析揭示显著差异 | 中 | 实验组比对照组差异超过 50% |
| 覆盖不全 | 原问题某个方面未被子问题覆盖 | 中 | 问"影响"但只分析了数量没分析质量 |
| 因果探索 | 发现相关性但未探索因果 | 低 | A 和 B 同时变化，是否有因果关系 |

### ⚠️ 原始问题未回答 — 最高优先级

回顾用户原始问题，检查当前分析是否真正回答了它。常见偏离场景：

| 偏离类型 | 示例 | 处理 |
|----------|------|------|
| 路径失败后放弃 | Funding 无 RU 数据就不分析资金了 | 尝试 Author→Org→Funding 等替代路径 |
| 话题偷换 | 问资金关系，最后只分析了出版物分布 | 重新从原始问题出发构造子问题 |
| 部分回答 | 只回答了"是什么"没回答"为什么" | 补充因果/机制分析 |

处理流程：
1. 重新审视实体映射，是否有未尝试的候选映射
2. 重新审视图拓扑，是否可通过多跳路径间接回答
3. 构造替代子问题逼近原始问题
4. 穷尽所有路径后仍无法回答 → 报告中明确说明原因和替代分析

### ⚠️ 可查的局限/建议 — 必须转化为子问题

4.7 递归总结中，每个局限/建议/后续方向都已评估是否"可转化为子问题"。对标记为"可查"的项目：

```
总结中写了: "建议分析俄罗斯作者的合作网络"
    → Schema 中有 coauthor 边？ → 有 → 生成新子问题 → 必须执行

总结中写了: "大量出版物缺少 FOS 标签"
    → 可以查这些出版物的年份/来源分布？ → 可以 → 生成新子问题 → 必须执行

总结中写了: "数据只覆盖到2024年"
    → 数据库本身的限制，无法查询更多 → 不可查 → 仅文字说明
```

### 第二层映射处理

在 Step 2 实体映射阶段，候选映射被分为两层：
- 第一层进入了初始拆解
- **第二层中分析价值较高的映射**被标记为"待深入探索"

处理流程：
1. 回顾 Step 2 标记的第二层映射
2. 结合第一轮分析结果，判断这些映射是否仍有补充价值
3. 对仍有价值的映射，生成新的子问题
4. 按 Step 3-4 的完整流程处理（查询 → 结论 → 可视化 → 对照）

## 2. 深入方向决策

对每个触发的检查项，决定深入方向：

### 2.1 追因分析

发现了"是什么"，追问"为什么"：

```
发现: Top10学校的平均成绩 85 分，远超全体平均 65 分
    │
    ▼
追因: Top10学校在哪些维度上不同？
    ├── 师资配比？
    ├── 生源质量？
    └── 资源投入？
```

### 2.2 细化分析

发现了整体趋势，需要更细粒度的洞察：

```
发现: 计算机科学领域发表量下降 38.8%
    │
    ▼
细化: 哪些子方向下降最严重？
    ├── AI/ML 方向
    ├── 系统方向
    └── 理论方向
```

### 2.3 关联分析

发现两个维度的变化可能相关：

```
发现: 国际合作减少 + 发表量下降
    │
    ▼
关联: 国际合作多的机构 vs 少的机构，发表量变化是否不同？
```

### 2.4 备选映射探索

高置信度映射已分析，尝试中/低置信度映射获取不同视角。

## 3. 执行流程

```
基础分析 + 对照分析完成
    │
    ▼
逐项执行深入检查（第1节的检查表）
    │
    ├── 无深入点 → 结束，进入报告撰写
    │
    └── 发现深入点
         │
         ▼
    确定深入方向（第2节）
         │
         ▼
    构造新的子问题
         │
         ▼
    回到主流程 Phase 2-5 执行
         │
         ▼
    将深入分析结果追加到报告中
         │
         ▼
    再次检查（最多迭代 2 轮）
```

## 4. 迭代控制

- **最大迭代深度**: 2 轮深入分析（防止无限发散）
- **终止条件**: 无新的有价值分析点 / 已达迭代上限 / 新分析未产生有意义的新发现
- **优先级**: 原始问题未回答 > 可查的局限/建议 > 第二层映射 > 异常值 > 意外发现 > 覆盖不全 > 因果探索

## 5. 报告格式

深入分析的结果作为报告的额外章节追加，与基础分析使用相同的 X.Y.1/X.Y.2/X.Y.3 格式：

```markdown
## 5. 深入分析

### 5.X [深入分析标题]

**触发原因**: [引用前面的具体发现]

#### 5.X.1 查询分析
[... 查询 → 结论 → 可视化 ...]

#### 5.X.2 对照验证
[... 对照设计 → 对比 → 结论 ...]

#### 5.X.3 小结
[综合结论]

### 5.Final 深入分析总结

**新发现**:
1. [新发现1]
2. [新发现2]

**与基础分析的关联**:
- [如何补充或修正基础分析的结论]
```

## 6. 一致性检查

深入分析完成后，必须检查最终报告的"局限性"章节：
- **禁止**将已经分析过的内容仍列为局限（如深入分析已做了学科分析，则"未区分学科"不能再出现）
- 局限性只包含确实无法通过当前数据回答的方向
