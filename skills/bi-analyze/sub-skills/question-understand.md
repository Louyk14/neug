# 问题理解 (Question Understanding)

> 本文件是 SKILL.md Step 2-3 的详细补充指南。核心规则（实体对齐表格式、拆解策略、递归规则）已在 SKILL.md 中内联，此处提供详细的实体映射方法论和拆解策略。

将用户的自然语言问题转化为可执行的分析计划：先做全面的实体映射，再做问题拆解。

## Part 1: 实体映射

### 1.1 问题解析

1. 解析用户原始问题，明确分析目标
2. 识别问题类型（分布分析、趋势分析、对比分析、相关性分析等）
3. 提取问题中的**所有关键实体**和约束条件

### 1.2 全面候选映射（每实体最多 10 个属性）

**⚠️ 核心要求**: 对问题中的每个实体，**尽可能全面**地考虑所有可能相关的 Schema 属性，每个实体最多列出 10 个候选映射。不要只列最明显的 1-2 个，要从多个角度思考实体可能对应的属性。

**映射思考维度**:

| 思考角度 | 说明 | 示例 |
|----------|------|------|
| 直接语义匹配 | 属性名直接对应实体含义 | "学校" → School.name |
| 标识符映射 | 实体的唯一编码或 ID | "学校" → School.CDSCode |
| 度量角度 | 实体相关的统计/度量属性 | "学术影响" → Publication.NumCited |
| 分类角度 | 实体可能是某个分类属性的值 | "物理学" → Publication.field_of_study |
| 关系角度 | 通过边连接到的其他节点属性 | "合作" → coauthor 边的存在性 |
| 聚合角度 | 需要对某属性做 COUNT/SUM | "发表量" → COUNT(Publication.id) |
| 约束角度 | 实体是某个约束条件的值 | "俄罗斯" → Organization.country = "Russia" |
| 时间角度 | 实体涉及时间维度 | "战后" → Publication.year >= 2022 |
| 间接关联 | 通过多跳路径关联的属性 | "学术产出" → Author → Publication → COUNT |
| 隐含属性 | 属性名中隐含了实体含义 | "高中" → NumOfHighSchool 已隐含学校类型 |

### 1.3 候选映射表格式

**⚠️ 必须在报告中输出**，每个候选映射都要说明**如何关联到实体**以及**能从什么角度回答问题**:

| 问题实体 | 候选映射 | 置信度 | 字段类型 | 关联方式 | 回答角度 |
|----------|----------|--------|----------|----------|----------|
| 学术影响 | Publication.NumCited | 高 | int32 | 被引次数直接衡量论文影响力 | 从引用指标角度回答影响力 |
| 学术影响 | COUNT(Publication.id) | 高 | measure | 统计论文数量 | 从产出数量角度回答影响力 |
| 学术影响 | Publication.field_of_study | 中 | string | 按领域分析影响力分布 | 从学科分布角度回答影响力 |
| 学术影响 | Author→Publication 边数量 | 中 | measure | 作者的论文产出 | 从个体产出角度回答影响力 |
| 学术影响 | Publication.journal_rank | 低 | string | 发表期刊的等级 | 从发表质量角度回答影响力 |

**置信度标准**:
- **高**: 语义完全匹配，这个属性直接回答实体含义
- **中**: 语义相关，能从某个侧面回答问题
- **低**: 间接相关，需要组合其他信息才能回答

### 1.4 约束条件提取

从问题中识别隐含和显式的约束条件：

- 显式约束: "成绩大于 80"
- 隐含约束: "高中学校" → 学校类型 = 高中
- 范围约束: "前 10 名" → top_n = 10
- 时间约束: "战后" → year >= 2022

## Part 2: 问题拆解

### ⚠️ 拆解与映射的分层策略

实体映射完成后，将候选映射按优先级分为两层：

**第一层（纳入初始拆解）**: 选择每个实体**最可能的几个映射**（高置信度 + 部分中置信度），据此生成子问题。这些是主要分析方向。

**第二层（留给深入分析）**: 剩余的中/低置信度映射中，如果有**分析价值较高**的（即能从不同角度补充回答原问题），标记为"待深入探索"。在第一轮子问题处理完毕后，进入 SKILL.md 4.8 深入分析判断，将这些映射生成新的子问题。

**判断"分析价值较高"的标准**:
- 提供了与第一层不同的分析视角
- 可能揭示第一层分析未覆盖的方面
- 对回答原问题有补充意义

**示例**:

```
实体"学术影响"的 10 个候选映射:

第一层（纳入初始拆解）:
  ✅ Publication.NumCited (高) → Q1: 从引用角度分析
  ✅ COUNT(Publication.id) (高) → Q2: 从发表量角度分析
  ✅ Publication.field_of_study (中) → Q3: 从学科分布角度分析

第二层（留给深入分析）:
  🔄 Author→Publication 边数量 (中) → 分析价值高，可补充个体层面视角
  🔄 Publication.journal_rank (低) → 分析价值高，可补充质量维度
  ✖ Publication.abstract (低) → 分析价值低，文本属性难以量化
  ✖ Publication.doi (低) → 标识符，无分析价值
```

### 拆解策略

**策略 A — 基于实体映射的多种可能性**

当实体存在多个候选映射时，按映射分类讨论：

```
原问题: "哪些学校成绩最好？"
├── Q1: 按 AvgScore 排名（高置信度映射）
├── Q2: 按 TestScore 排名（中置信度映射）
└── Q3: [总结] 综合 Q1、Q2 回答原问题
（第二层: NumGE1500 映射留给深入分析）
```

**策略 B — 基于问题涉及的不同方面**

当问题涉及多个分析维度时，按方面拆分：

| 拆解维度 | 示例 |
|----------|------|
| 按方面 | 学校分析 → 成绩、师资、设施 |
| 按步骤 | 找最好的学校 → 定义指标 → 计算 → 排序 |
| 按维度 | 分布分析 → 按地区、按规模、按类型 |
| 按时间 | 当前状态 → 历史趋势 |
| 按层级 | 整体概览 → 分类细分 → 个体详情 |

**策略选择**:

| 场景 | 推荐策略 | 原因 |
|------|----------|------|
| 实体映射不确定 | 策略 A | 需要探索不同映射 |
| 问题涉及多维度 | 策略 B | 需从不同角度分析 |
| 两者兼有 | A + B 组合 | 先按映射分类，再按维度细分 |

### 递归拆解

子问题如果仍然复杂，可以继续拆解（不同层级可用不同策略）。

**示例: 两级拆解**

```
原问题: "俄乌冲突对学术有什么影响？"
│
├── Q1: 发表量变化 ← 仍复杂，继续拆
│   ├── Q1.1: 按学科领域分析
│   ├── Q1.2: 按机构类型分析
│   └── Q1.3: [总结] 综合回答 Q1
│
├── Q2: 国际合作变化 ← 直接可查
├── Q3: 资助来源变化 ← 直接可查
└── Q4: [总结] 综合 Q1-Q3 回答原问题

深入分析（第一轮完成后）:
├── Q5: 引用影响力变化（第二层映射: NumCited）
├── Q6: 发表期刊等级变化（第二层映射: journal_rank）
└── Q7: [总结] 综合 Q5-Q6 补充原分析
```

### 拆解规则

1. 每次拆解必须**说明策略和原因**
2. 同级子问题完成后必须增加**总结小节**
3. 拆解深度一般不超过 3 级
4. 对于每个子问题，每个实体只选择一个映射
5. **第二层映射不丢弃**，标记后留给 4.8 深入分析

### 拆解报告格式

每次拆解在报告中记录：

```markdown
### X.Y [问题标题]

**拆解策略**: [策略 A / 策略 B / 组合]
**拆解原因**: [选择此策略的原因]

**子问题**:
1. X.Y.1 [子问题1描述]
2. X.Y.2 [子问题2描述]
3. X.Y.Z [总结] 综合以上子问题回答 X.Y

**待深入探索的映射**（留给 4.8 深入分析）:
- [映射A]: [分析价值说明]
- [映射B]: [分析价值说明]
```

## 产出

1. **实体对齐表**: 所有实体的全部候选映射（每实体最多 10 个，含置信度、关联方式、回答角度）
2. **映射分层**: 明确哪些进入第一层拆解，哪些留给深入分析
3. **子问题树**: 递归拆解后的完整子问题结构（含拆解策略说明）
4. **每个叶子子问题**: 明确的实体映射选择 + 分析目标
