# 报告后处理（修复 subagent 异步导致的报告问题）

> **仅顶层 agent（depth=0）使用**。在按 report-gen.md 组装报告之后执行此修复流程。
>
> subagent（depth>0）不需要执行此文件，subagent 只需将分析结果保存到指定路径即可。

---

## 为什么需要此步骤

subagent 异步执行追因分析后，报告中常见以下问题：

1. **占位符残留**: 主 agent 在等待 subagent 时写入了占位文本
2. **内容错位**: subagent 输出被追加到报告末尾，而非插入到 4.X.3 位置
3. **标题级别错误**: subagent 独立生成的内容用了 `#`/`##` 等顶层标题
4. **文本连接异常**: 内容追加时缺少换行，两段文本粘连在一行
5. **小结未更新**: 4.X.4 小结仍引用占位符而非追因实际结论

---

## 修复流程

### Step 1: 收集 subagent 输出

读取所有 subagent 保存的追因分析文件。记录每个文件对应哪个子问题的 4.X.3。

### Step 2: 清除占位符

在报告中搜索并删除以下模式的文本：

```
❌ "详见 subagent 生成的追因分析报告"
❌ "由 subagent 执行中，将补充查询验证"
❌ "追因分析待 subagent 完成后补充"
❌ "（已保存）"
❌ 任何包含 "subagent" 字样的占位说明
```

### Step 3: 移动错位内容

检查报告末尾（附录之后）和各 4.X.4 之后是否存在错位的追因内容：

**检测信号**:
- 在 `#### 4.X.4 小结` 之后出现了 `4.X.3.Y` 编号的内容
- 在 `## 5. 结论` 或 `## 附录` 之后出现了分析内容
- 出现独立的 `## 4.X.3 为什么...` 标题（标题级别明显过高）

**修复**: 将这些内容剪切，移动到对应的 `#### 4.X.3 追因分析` 章节内。

### Step 4: 修正标题级别

subagent 输出中的标题需要降级到正确层级：

| subagent 输出 | 应改为 | 原因 |
|--------------|--------|------|
| `# 标题` | `##### 4.X.3.Y 标题` | 追因子问题是 h5 |
| `## 标题` | `##### 4.X.3.Y 标题` | 同上 |
| `### 子标题` | `###### 子标题` | 追因子问题内的内容是 h6 |

具体级别取决于追因深度：
- depth=1 的追因子问题: `#####` (h5)
- depth=2 的追因子问题: `######` (h6)

### Step 5: 修复文本连接

逐行扫描报告，检查是否有一行内包含两段不相关的内容（通常是前一段的最后一行和 subagent 输出的第一行粘连）。在正确位置插入空行。

### Step 6: 更新 4.X.4 小结

对每个子问题，确认 4.X.4 小结中的**追因结论**引用的是 subagent 的实际分析结论（含数值），而非占位文本。

### Step 7: 全文通读验证

从报告第一行开始逐段检查：

- [ ] 章节编号连续无跳跃（1→2→3→4.1→4.1.1→...→4.1.4→4.2→...）
- [ ] 无占位符文本残留（搜索 "subagent"/"已保存"/"执行中"）
- [ ] 追因内容位于 4.X.3 内部（不在 4.X.4 之后、不在报告末尾）
- [ ] 标题级别正确递进（## → ### → #### → ##### → ######）
- [ ] 无文本粘连（每个标题独占一行，标题前有空行）
- [ ] 每个 4.X 完整包含 .1(查询) + .2(对照) + .3(追因) + .4(小结)
- [ ] 图片路径统一为 `images/` 相对路径
- [ ] 执行摘要包含追因发现（不只是主分析数据）
